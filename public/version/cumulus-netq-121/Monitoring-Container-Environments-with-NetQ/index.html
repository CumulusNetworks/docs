Monitoring Container Environments with NetQ
This documentation is for an older version of the software. If you are using the current version of Cumulus Linux, this content may not be up to date. The current version of the documentation is available [here](https://docs.cumulusnetworks.com/).

Monitoring Container Environments with NetQ

The NetQ Agent monitors Docker and Mesos Universal Container Runtime containers the same way it monitors physical servers. There is no special implementation. The NetQ Agent pulls Docker data from the container as it would pull data from a Cumulus Linux switch or Linux host.

For more information, see the Host Pack user guide.
